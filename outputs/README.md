# Outputs Directory

This directory contains generated artifacts produced by OpenAMP Foundry pipeline runs. All artifacts are **dry-lab only** — computational outputs that require independent wet-lab validation before any biological conclusions can be drawn.

## What is in this directory

| File pattern | Description |
|---|---|
| `bench_*.json` | Benchmark results (precision@k, calibration, per-family) |
| `batch_*.fasta` | FASTA exports of candidate sequences for partner review |
| `calibration_audit_example.json` | Example calibration audit output |
| `baseline_trivial.json` | Trivial baseline scores for cheap-enemy comparison |
| `*.json` | Pipeline run outputs — see schema documentation |

## Dry-lab-only notice

**None of these outputs constitute biological proof.** They represent computational nominations based on dry-lab scoring pipelines. The pipeline does not predict in-vivo efficacy, toxicity, or any wet-lab outcome.

Terms that apply to outputs in this directory:
- "computationally nominated"
- "dry-lab candidate"
- "selected for review"
- "evidence package"

Terms that do NOT apply:
- "validated"
- "confirmed"
- "proven effective"
- "biologically active"

## What is git-ignored

Most pipeline run outputs are listed in `.gitignore` and are NOT committed. This directory contains only curated example outputs that are small, safe, and useful for reviewers and new contributors to understand what good output looks like.

To regenerate these examples: `make demo`

## Interpreting outputs

Each output file is generated by a schema in `src/openamp_foundry/`. The corresponding test file shows what fields are required and what validation rules apply.

For evidence certificates (CERT- prefix) and related records, see `docs/evidence/` for interpretation guidance.

For benchmark outputs, see `docs/BENCHMARK_CAVEATS.md` before drawing conclusions.
