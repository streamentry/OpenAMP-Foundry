# Public Readiness Guide

How to determine if the project is ready for public sharing.

## Checks
- [ ] Release readiness checks pass
- [ ] Claim wording is safe (`check_claims.py`)
- [ ] LIMITATIONS_OVERVIEW.md is accurate
- [ ] PUBLICATION_PACK.md checklist is complete
- [ ] README.md is up to date
- [ ] CITATION.cff is up to date

## Levels of Public Readiness
| Level | Meaning | Share With |
|-------|---------|-----------|
| Internal | Project team only | Team members |
| Review-only | Ready for expert review | External reviewers |
| Public-safe | Ready for public | Anyone |
| Release | Ready for publication | General public |
