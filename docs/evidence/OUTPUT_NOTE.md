# Output Note

OpenAMP Foundry should make outputs easy to inspect and compare.

```markdown
## Output note

**Output path:** File or directory.

**Created by:** Command, workflow, or script.

**Inputs:** Configs or upstream files.

**Purpose:** Why this output exists.

**Review status:** unchecked | checked | review-needed | superseded.

**Next action:** keep | regenerate | remove after review.
```

## Acceptance test

An output note is useful when another contributor can tell why an output exists, how it was produced, and whether it should be trusted, regenerated, or removed.
