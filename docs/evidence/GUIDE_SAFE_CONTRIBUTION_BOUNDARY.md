# Safe Contribution Boundary Guide

What contributions are safe to make without maintainer discussion,
and what requires discussion first.

## Safe (No Discussion Needed)
- Typo fixes and formatting improvements
- Adding tests for existing functionality
- Adding documentation for existing features
- Updating example data or demo files

## Discussion Recommended
- Adding new features or modifying existing behavior
- Adding new documentation categories
- Changing test infrastructure
- Adding new dependencies

## Discussion Required
- Changes to safety policy
- Changes to scoring weights
- Changes to calibration pipeline
- Any change that affects claims or releases
- Adding new benchmarks that affect the gate
