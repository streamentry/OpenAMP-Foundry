# Generated-File Warning Standard

Files that are generated by scripts or CI must carry a warning.

## Warning Format

```markdown
> **NOTE:** This file is generated. Do not edit directly.
> Source: [path to source script or config]
> Regenerate with: `make [target]`
```

## Where to Apply
- Benchmark output files (if committed)
- Generated documentation
- Auto-generated configs
- CI-generated artifacts

## Exceptions
- Files that are both source and output (e.g., README.md that is also generated)
- Files in `outputs/` which are git-ignored
