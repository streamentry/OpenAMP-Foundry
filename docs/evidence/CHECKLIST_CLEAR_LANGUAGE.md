# Clear-Language Review Checklist

- [ ] Avoid jargon where plain language works
- [ ] Define acronyms on first use
- [ ] Use active voice ("the pipeline scores candidates" not "candidates are scored")
- [ ] Keep sentences under 25 words where possible
- [ ] Use bullet points for lists of 3+ items
- [ ] Use numbered steps for procedures
- [ ] Avoid "easily," "simply," "obviously" — these hide complexity
- [ ] Run `check_claims.py` to catch overclaiming language
