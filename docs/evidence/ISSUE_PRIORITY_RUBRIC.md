# Issue Priority Rubric

How to assign priority to issues.

| Priority | Label | Response Time | Example |
|----------|-------|:-------------:|---------|
| P0 | critical | < 24h | Safety policy error, broken CLI |
| P1 | high | < 72h | Benchmark regression, broken test |
| P2 | medium | < 1 week | Missing feature, doc gap |
| P3 | low | Best effort | Nice-to-have improvement |
