# Scenario Playbook: Large PRs

**Scenario:** A PR exceeds 300 lines and touches multiple areas.

## Steps

1. Request the author to split the PR into smaller, focused PRs.
2. Each smaller PR should address one concern (e.g., feature change, bug fix, docs update).
3. If splitting is not practical (e.g., a coordinated change across files):
   - Request a design document or RFC first.
   - Assign multiple reviewers for different areas.
   - Set a longer review timeline.
4. Ensure the PR description explains why the large size is necessary.
