# Contributor FAQ

## How do I get started?
Read CONTRIBUTING.md, then QUICKSTART.md. Pick an issue from the open issues list.

## What makes a good first issue?
Issues labeled `good-first-issue` are ideal. They're small, well-scoped, and don't require deep project knowledge.

## How do I submit a PR?
1. Create a branch from main.
2. Make your change and add tests.
3. Run `python3 -m pytest -q` to verify.
4. Run `python3 scripts/check_doc_links.py` to check links.
5. Submit the PR referencing the issue number.

## What if I get stuck?
Ask in the issue comments. If it's urgent, tag a maintainer.

## Can I contribute without writing code?
Yes! Documentation improvements, test additions, and issue triage are all valuable.
