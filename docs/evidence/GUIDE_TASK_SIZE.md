# Contributor Task Size Guide

How to estimate task size for issues.

| Size | Lines of Change | Review Time | Example |
|:----:|:---------------:|:-----------:|---------|
| Small | < 50 | < 30 min | Typo fix, single doc update |
| Medium | 50-200 | 1-2 hours | New doc, small feature, test addition |
| Large | 200-500 | 2-4 hours | New script, policy document |
| XLarge | > 500 | > 4 hours | Major feature, architecture change |

## Rules
- Large or XLarge tasks should be broken into smaller issues.
- If an issue is XLarge, create a design doc first.
- Task size should be estimated before starting work.
