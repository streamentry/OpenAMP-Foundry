# Urgent Documentation Correction Policy

For critical errors that could mislead readers or cause harm.

## What Qualifies as Urgent

- Safety policy misstatement
- Incorrect assay protocol
- Overclaiming language about biological proof
- Broken link in a critical path (quickstart, safety policy)

## Process

1. Fix the error immediately.
2. Open a PR with the `urgent` label.
3. At least one maintainer must review within 24 hours.
4. After merge, log the correction in the decision log.

## Post-Correction

- If the error was present in a released version, issue a correction notice.
- Review what allowed the error to pass review and adjust process.
