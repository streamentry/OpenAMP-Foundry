# Duplicate Issue Handling Guide

How to identify and handle duplicate issues.

## Steps

1. Before creating a new issue, search for existing issues with similar keywords.
2. If a duplicate is found, comment on the new issue linking to the original.
3. Close the duplicate with a note referencing the original issue number.
4. If the duplicate contains unique information, add it as a comment to the original issue before closing.

## Prevention

- Maintain a clear issue label system (see ISSUE_LABEL_GUIDE.md).
- Review open issues regularly for potential duplicates.
