# Documentation Escalation Policy

When a documentation issue cannot be resolved at the contributor level, it escalates.

## Levels

| Level | Who | Scope | Example |
|-------|-----|-------|---------|
| 1 | Contributor | Own changes | Fixing a typo |
| 2 | Reviewer | PR scope | Resolving a doc structure disagreement |
| 3 | Maintainer | Project scope | Policy change affecting all docs |
| 4 | Project lead | Cross-project | License or safety doc changes |

## Rules

- Escalate only after attempting resolution at the current level.
- Document the escalation in the PR or issue comments.
- Higher levels can delegate back to lower levels.
