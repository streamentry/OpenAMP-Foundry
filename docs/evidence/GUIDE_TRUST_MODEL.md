# Trust Model Overview

How trust is established in the OpenAMP Foundry project.

## Trust Layers
1. **Code transparency** — all code is open source (Apache 2.0)
2. **Evidence transparency** — all results include caveats and limitations
3. **Process transparency** — decisions are logged in decision logs
4. **Reproducibility** — all outputs can be regenerated from versioned inputs
5. **Review** — expert review is required before release

## What Trust Does NOT Mean
- Trust in the pipeline does not mean trust in the predictions as biological proof.
- Trust in the code does not mean the code is bug-free.
- Trust in the process does not mean the process guarantees correct results.
