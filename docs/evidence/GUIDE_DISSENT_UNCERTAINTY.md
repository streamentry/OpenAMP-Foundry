# Dissent and Uncertainty Guide

How to handle disagreement and uncertainty in reviews.

## Dissent
- A reviewer may dissent from the majority opinion.
- Dissent must be documented in the review comments.
- Dissent does not block the PR unless the reviewer has blocking authority.
- Dissent should include the reasoning and, if possible, an alternative proposal.

## Uncertainty
- If a reviewer is uncertain about a change, they should ask for clarification.
- If uncertainty cannot be resolved, escalate to a maintainer.
- Uncertainty should not block a change indefinitely — set a deadline for resolution.

## Recording
All dissents and significant uncertainties should be logged in the decision log.
