# Issue Triage Matrix

How to triage incoming issues.

| Factor | High Priority | Medium Priority | Low Priority |
|--------|:-------------:|:---------------:|:------------:|
| Safety impact | Direct safety concern | Indirect safety concern | No safety impact |
| User impact | Blocks all users | Blocks some users | Minor inconvenience |
| Effort | Small fix (< 1 day) | Medium fix (1-3 days) | Large fix (> 3 days) |
| Alignment | Core mission | Supporting | Nice-to-have |

## Process
1. Apply appropriate labels (bug, enhancement, docs, safety).
2. Assign priority (P0-P3).
3. If P0, notify maintainers immediately.
4. If good-first-issue, add the label.
