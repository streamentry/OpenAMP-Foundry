# No-Hype Writing Rules

Rules for keeping documentation honest and grounded.

1. **No superlatives without evidence.** Don't say "best" or "first" unless you can prove it.
2. **No claims about biology without wet-lab data.** Computational results are predictions, not proof.
3. **State limitations as prominently as findings.** Every claim should be paired with its caveat.
4. **Use the weakest accurate claim.** If "suggests" is accurate, don't use "demonstrates."
5. **No future capability as current fact.** "Will be able to" is speculation, not documentation.
6. **No hidden negatives.** Failed experiments, benchmark weaknesses, and model limitations must be documented.
7. **Review before publishing.** Run `check_claims.py` on all public-facing text.
