# Metric Guide: Contributor Docs

**Question:** Does a new contributor have everything they need to make a first change?

**What to track:** Whether all contributor-facing docs cover the full path from setup to merged PR.

**Documents to check:**
- CONTRIBUTING.md
- QUICKSTART.md
- CONTRIBUTOR_CURRICULUM.md
- RUNBOOK_PICKING_UP_ISSUE.md
- RUNBOOK_ASKING_FOR_REVIEW.md

**Target:** A contributor can follow the documented path without gaps.

**What good looks like:** A first-time contributor can clone, run the demo, pick an issue, make a change, and submit a PR using only the documented steps.
