# Issue Dependency Guide

How to track and manage dependencies between issues.

## Identifying Dependencies

- Issue A depends on Issue B if A cannot be started until B is complete.
- Use the "depends on" feature in GitHub Issues.
- Document the dependency in both issues' descriptions.

## Managing Dependencies

- Order dependent issues in the same milestone.
- If a dependency blocks a milestone, escalate it.
- If a dependency is itself blocked, consider breaking it into smaller issues.
- Remove the dependency if the issues can be worked on independently.
