# Reader Self-Assessment Checklist

Use this checklist to assess your understanding of the project documentation.

## Level 1: Can run the pipeline
- [ ] I can run `make demo` and understand the output
- [ ] I can find evidence certificates in `outputs/evidence/`
- [ ] I know what `charge_bias` means
- [ ] I understand that scores are computational, not biological proof

## Level 2: Can contribute
- [ ] I know how to find the right document from PROJECT_INDEX.md
- [ ] I can run docs checks locally
- [ ] I understand the contribution workflow from CONTRIBUTING.md
- [ ] I know how to create a PR referencing an issue

## Level 3: Can review
- [ ] I can assess a PR for overclaiming language
- [ ] I understand the benchmark limitations
- [ ] I know where to find the current metrics
- [ ] I understand the simulation module status
