# Docs Freshness Policy

Documents must be reviewed and updated on a regular cadence.

| Document Type | Maximum Age | Review Trigger |
|---------------|:-----------:|----------------|
| Policy docs | 6 months | Calendar reminder |
| Getting-started guides | 3 months | After CLI changes |
| Evidence docs | 6 months | After benchmark changes |
| Concept cards | 12 months | After policy changes |
| Runbooks | 3 months | After workflow changes |
| Audit checklists | 6 months | Before release |

## Process
1. Reviewer checks accuracy, completeness, links.
2. Outdated docs are updated or flagged for retirement.
3. Review results recorded in decision log.
