# Command Note

OpenAMP Foundry should make important command outputs easy to inspect later.

```markdown
## Command note

**Name:** Short name.

**Date or commit:** Date, version, or commit.

**Command or workflow:** Command, script, or workflow name.

**Inputs:** Configs and relevant artifacts.

**Outputs:** Files or reports produced.

**Known limits:** Missing checks or uncertainty.

**Next check:** The smallest useful follow-up.
```

## Acceptance test

A command note is useful when another contributor can repeat or challenge the work without guessing hidden context.
