# Release Notes Writing Guide

How to write release notes.

## Structure
```
## v0.5.N — Title

Summary of the release in 1-2 sentences.

### Added
- New feature or document

### Changed
- Existing feature or document update

### Fixed
- Bug fix

### Deprecated
- Feature or document no longer recommended

### Removed
- Feature or document removed
```

## Rules
- Use the change note format (file path + description).
- Group changes by type (Added, Changed, Fixed).
- Include a summary sentence at the top.
- Link to relevant issues or PRs.
- Avoid overclaiming language.
