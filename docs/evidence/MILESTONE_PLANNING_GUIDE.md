# Milestone Planning Guide

How to plan and manage milestones.

## Milestone Structure

| Milestone | Duration | Focus |
|-----------|:--------:|-------|
| v0.6.x | 2 weeks | Bug fixes, small improvements |
| v0.7.x | 4 weeks | Feature work |
| v0.8.x | 4 weeks | Integration, testing |
| v1.0 | TBD | Release candidate |

## Planning Process

1. Review open issues and group by area.
2. Estimate effort (small/medium/large).
3. Select issues for the milestone based on priority and capacity.
4. Assign issues to the milestone in GitHub.
5. Review progress at milestone midpoint.
