# Artifact Lifecycle Guide

The lifecycle of a pipeline artifact from creation to archive.

## Stages
1. **Created** — generated by a pipeline run or script
2. **Validated** — checked against its schema
3. **Used** — consumed by downstream processes (reports, calibration)
4. **Reviewed** — included in a review packet or release
5. **Archived** — stored for historical reference after its active use

## Rules
- Every artifact should be validated at creation time.
- Artifacts should not be modified after creation (immutable).
- If an artifact needs to change, create a new version.
- Artifacts used in releases should be archived with the release.
