# Documentation Quality Checklist

Use this to assess documentation quality.

## Structure
- [ ] Document has a clear purpose stated at the top
- [ ] Headings follow a logical hierarchy
- [ ] Related documents are linked

## Content
- [ ] No overclaiming language (run `check_claims.py`)
- [ ] All links resolve (run `check_doc_links.py`)
- [ ] Examples work from a clean checkout
- [ ] Limitations are stated where relevant

## Maintenance
- [ ] Document is no more than 6 months old
- [ ] Document is indexed in PROJECT_INDEX.md
- [ ] Document has an identified owner
