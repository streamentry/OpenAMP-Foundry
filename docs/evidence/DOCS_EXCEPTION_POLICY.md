# Documentation Exception Policy

An exception allows a deliberate deviation from a documentation standard.

## When to Use

- A document must be published before all standards can be met.
- A legacy document cannot be updated without breaking external references.
- A temporary situation requires faster publishing than full compliance allows.

## Requirements

- The exception must be documented in the decision log.
- The exception must include a reason and an expiry date.
- The exception must be reviewed at expiry.
