# External Reviewer Onboarding Note

When onboarding an external reviewer, provide:

## Required
- Project README.md for context
- LIMITATIONS_OVERVIEW.md for honest caveats
- The specific artifact or claim being reviewed
- The review questionnaire

## Optional
- METRICS_CURRENT.md for benchmark context
- ARCHITECTURE.md for system design
- Relevant concept cards

## What NOT to Share
- Unpublished candidate lists before review is complete
- Sensitive or dual-use information without clear need
