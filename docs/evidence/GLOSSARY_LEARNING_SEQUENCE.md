# Glossary Learning Sequence

Recommended order to learn project terminology.

1. **Evidence** — what constitutes proof at different levels (PROOF_LADDER.md)
2. **Artifact** — any generated file that carries evidence (ARTIFACT_MAP.md)
3. **Schema** — machine-readable definition of an artifact's structure
4. **Manifest** — record of inputs, configs, and outputs for a run
5. **Gate** — a safety mechanism that blocks or allows an action
6. **Abstention** — when a module declines to make a prediction due to uncertainty
7. **Baseline** — a simple heuristic that any new method must beat
8. **Calibration** — adjusting model outputs based on real data
