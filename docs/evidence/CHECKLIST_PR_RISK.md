# Pull Request Risk Checklist

Use this to assess PR risk before review.

## Questions
- [ ] Does the PR change scoring weights? (High risk)
- [ ] Does the PR change calibration pipeline? (High risk)
- [ ] Does the PR change safety policy? (High risk)
- [ ] Does the PR add new dependencies? (Medium risk)
- [ ] Does the PR change CLI output format? (Medium risk)
- [ ] Does the PR add new functionality? (Medium risk)
- [ ] Does the PR fix a bug? (Low risk)
- [ ] Does the PR improve documentation? (Low risk)

Based on answers, assign the appropriate review depth per the PR review rubric.
