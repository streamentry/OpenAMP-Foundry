# Artifact — Concept Card

An artifact is any file generated by the pipeline that carries evidence,
a score, or a decision record.

## Types

| Type | Examples | Must be versioned? |
|------|----------|:------------------:|
| Input | Candidate CSV, config YAML | Yes |
| Intermediate | Ranked JSONL, evidence certs | Yes |
| Report | Batch report, benchmark output | Yes |
| Manifest | Run manifest, batch manifest | Yes |
| Review | Review request, decision log | Yes |

## Rules

- Every artifact should have a defined schema or format.
- Artifacts that affect decisions must be reproducible.
- Temporary artifacts (caches, logs) are excluded.
