# Safe Filename Policy

Filenames in the repository must follow these rules.

## Allowed Characters
- Lowercase letters: `a-z`
- Digits: `0-9`
- Hyphens: `-`
- Underscores: `_`
- Dots: `.`

## Rules
- No spaces in filenames.
- No special characters except `-` and `_`.
- Use `.md` for documentation, `.py` for Python, `.json` for JSON.
- Use snake_case for Python files and data files.
- Use lowercase for all filenames.

## Examples
- ✅ `check_doc_links.py`
- ✅ `METRICS_CURRENT.md`
- ❌ `My File.py`
- ❌ `data-file(v2).csv`
