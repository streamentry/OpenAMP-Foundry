# Docs Issue Template Guide

How to create and use issue templates for documentation tasks.

## Available Templates
| Template | Location | When to Use |
|----------|----------|-------------|
| Bug report | `.github/ISSUE_TEMPLATE/bug_report.md` | Something is broken |
| Documentation improvement | `.github/ISSUE_TEMPLATE/documentation_improvement.md` | Suggesting a doc change |
| Expert review | `.github/ISSUE_TEMPLATE/expert_review.yml` | Requesting expert review |
| Safety review | `.github/ISSUE_TEMPLATE/safety_review.md` | Safety concern |
| Model release review | `.github/ISSUE_TEMPLATE/model_release_review.md` | Model release request |
| Data contribution | `.github/ISSUE_TEMPLATE/data_contribution.md` | Contributing data |

## When to Use a Blank Issue
- The task doesn't fit any existing template.
- The task is exploratory and doesn't have a clear template yet.
- The task spans multiple template categories.
