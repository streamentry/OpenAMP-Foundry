# Documentation Governance FAQ

## Who owns the documentation?
The project maintainers own the documentation. Contributors are encouraged to
improve any document that helps them work.

## How are decisions about documentation made?
Major changes (new policies, restructuring) go through the standard PR and
review process. Minor changes (typos, formatting) can be committed directly.

## What happens when docs disagree?
`docs/evidence/METRICS_CURRENT.md` is the single source of truth for metrics.
For other conflicts, the most recently updated document takes precedence.

## Can I add a new document?
Yes. Add it to the appropriate subdirectory under `docs/` and link it from
`docs/PROJECT_INDEX.md`.

## How do I find the right document?
Start at `docs/README.md` (the front door) or `docs/PROJECT_INDEX.md`
(the exhaustive catalog).

## Who reviews documentation PRs?
Any maintainer can review documentation PRs. Safety-critical docs require
a safety reviewer.
