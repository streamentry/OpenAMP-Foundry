# Reviewer Curriculum

A staged learning path for code and documentation reviewers.

## Stage 1: Can review low-risk PRs
- Understand the difference between computational and biological evidence
- Can run `check_claims.py` and identify overclaiming language
- Can run `check_doc_links.py` and fix broken links
- Can verify PRs against the PR review rubric

## Stage 2: Can review medium-risk PRs
- Understand the benchmark limitations and caveats
- Can assess whether a PR's scope matches its description
- Can verify that tests pass and cover the changed code
- Can assess whether a PR is safe to merge

## Stage 3: Can review high-risk PRs
- Understand the calibration pipeline and gate
- Can assess safety policy implications
- Can verify that changes are backward-compatible
- Can assess whether a PR needs design review first
