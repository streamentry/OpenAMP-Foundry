# Public Summary Writing Guide

How to write summaries for public audiences.

## Structure
1. **What was done** — one sentence
2. **Key finding** — one sentence
3. **Limitations** — what the result does NOT prove
4. **Next steps** — what comes next

## Rules
- Use plain language, avoid jargon
- State limitations as prominently as findings
- Never claim biological proof without wet-lab data
- Use the project's allowed claim language
- Run `check_claims.py` before publishing

## Example
> "We computationally nominated 24 antimicrobial peptide candidates
> using an open, reproducible pipeline. These are predictions only —
> no antimicrobial activity has been demonstrated. Wet-lab validation
> is required before any biological claim can be made."
