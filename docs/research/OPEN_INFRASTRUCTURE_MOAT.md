# Open Infrastructure Moat

## Purpose

This document explains the defensible long-term value of OpenAMP Foundry.

The project should not try to win by claiming the best model forever.

Models change.

Trust infrastructure compounds.

## Prime thesis

**OpenAMP’s moat should be the evidence, safety, benchmark, release, and review layer around antimicrobial peptide candidate selection.**

That layer is harder to copy than a single score.

## What is not a durable moat

Weak moats:

- one predictor;
- one leaderboard score;
- one generator;
- one impressive candidate list;
- one viral demo;
- private maintainer intuition;
- aggressive claims.

These can be copied, beaten, or discredited.

## Durable moat candidates

Stronger moats:

- evidence certificate schema;
- benchmark governance;
- proof ladder;
- release policy;
- dataset and model cards;
- external review packet standard;
- negative-result archive;
- recalibration gate;
- run manifest standard;
- adapter interface discipline;
- human-agent collaboration rails;
- trust center and decision records.

These are coordination infrastructure.

## Why open infrastructure can win

Biotech discovery needs many actors:

- software engineers;
- computational scientists;
- peptide experts;
- microbiologists;
- safety reviewers;
- institutions;
- funders;
- labs;
- AI agents;
- downstream tool builders.

No single actor wants opaque control by another actor.

A neutral, auditable, safety-aware open layer can become the shared substrate.

## The Linux analogy, bounded

The analogy is not that OpenAMP is already Linux-scale.

The analogy is that durable infrastructure often wins by becoming:

- boring;
- stable;
- trusted;
- extensible;
- hard to replace;
- useful to competitors;
- governed by interfaces and norms.

OpenAMP should aim to be the boring trust layer underneath many AMP discovery workflows.

## The agent advantage

AI agents need rails.

A repo with clear schemas, source-of-truth docs, issue templates, proof ladders, and stop conditions lets agents contribute safely.

That creates a compounding advantage:

```text
better rails
  -> safer agent tasks
  -> more useful infrastructure PRs
  -> stronger artifacts
  -> easier external review
  -> more trust
  -> better rails
```

## The human advantage

Humans need evidence they can inspect.

OpenAMP should help serious humans answer:

- What happened?
- Why was this selected?
- Which baseline could explain it?
- What is the release status?
- What is the proof level?
- What remains unproven?
- What decision was made?

The repo that answers these fastest earns trust.

## Strategic implication

Prioritize work that becomes useful even if the current scorer is replaced.

High-priority examples:

- manifests;
- schemas;
- validators;
- benchmark cards;
- adapters;
- review packets;
- release decisions;
- claim checks;
- calibration gates;
- negative-result records.

Low-priority examples:

- impressive demo copy;
- model-only improvements without baselines;
- new scores without interpretation;
- candidate lists without review;
- broad claims without evidence.

## OpenAMP should be useful to competitors

The strongest infrastructure is useful even to groups that disagree with the model.

An external lab, company, or academic group should be able to use OpenAMP’s:

- evidence certificate format;
- dataset card standard;
- model card standard;
- benchmark governance;
- review packet;
- proof ladder;
- release checklist.

That is not weakness.

That is ecosystem position.

## Final standard

OpenAMP becomes hard to replace when outsiders use its process to make their own work more trustworthy.

That is the moat.
