# Contributor Runbook: Handling Review Feedback

1. Read all feedback before responding.
2. Separate into:
   - **Required changes** — must be addressed before merge.
   - **Suggestions** — optional improvements.
   - **Questions** — need clarification.
3. Make required changes, push to the same branch.
4. Reply to each comment indicating what you changed.
5. If you disagree with feedback, explain your reasoning respectfully.
6. After addressing all feedback, re-request review.
